BMC Group Dashboard Guide

How to read and use your monitoring dashboards

Documentation

Your two dashboards,
explained simply.

BMC Group runs on a connected set of services — from the website and CMS through to Stripe payments, email and fuel prices. These two dashboards give you visibility of everything in one place. This guide explains what each panel is checking, what good looks like, and what to do when something goes wrong.

Vercel / Website
Status Dashboard ▼
What it does
Checks that the BMC Next.js website is live and reachable. It pings three pages — the homepage, inventory listing, and finance page — to confirm all three are responding correctly. Vercel hosts the website and handles all deployments made by Elias.
Homepage
Main entry point — if this is down the whole site is down
Inventory
Vehicle listings page — most visited page on the site
Finance page
Codeweavers finance embed — critical for finance enquiries
What to look out for
All green — Operational
All three pages are responding. No action needed.
Amber — one page not responding
Check Vercel dashboard for deployment errors. May be a temporary Vercel outage — try refreshing in 2 minutes. If persisting, contact Elias.
Red — site completely unreachable
Check Vercel status page (vercel.com/status). If Vercel is up but site is down, contact Elias immediately — a deployment may have broken the site.
🗄
Sanity CMS
Status Dashboard ▼
What it does
Sanity is the content management system that stores all vehicle data, promotions, location information and site settings. The dashboard checks that both the Sanity API (which the website queries for content) and the Sanity Studio (where you manage content) are reachable.
What to look out for
Green — API and Studio both reachable
Content is flowing normally. Vehicle pages, promotions and locations are all live.
Amber — Studio unreachable but API working
The website will still work as it queries the API directly. You won't be able to edit content in the studio until it recovers. Check sanity-status.com.
Red — Sanity API down
The website may show blank vehicle pages or errors. This is a Sanity platform issue — check sanity-status.com and contact Sanity support if it persists beyond 15 minutes.
💳
Stripe Reservations
Status Dashboard ▼
What it does
Checks that the Stripe payment infrastructure is working. It verifies the Stripe API is reachable, the reservation webhook endpoint is live at /api/stripe-webhook, and the /api/reserve route that creates checkout sessions is responding. The webhook is critical — it's what marks a vehicle as reserved in Sanity after a payment completes.
What to look out for
Always amber until go live — Test mode active
This is expected. The dashboard will show amber because Stripe is in test mode using sk_test_ keys. No real payments are being processed. This will turn green after go live when live keys are activated.
Red — webhook endpoint or reserve API down
Customers will not be able to complete reservations. Contact Elias immediately. Check Vercel logs for errors on /api/stripe-webhook and /api/reserve.
Known: Currently in test modeThe Stripe integration is fully built and working but is using test keys (sk_test_). Real payments will only be taken after go live when live keys (sk_live_) are activated in Vercel by Ant.

Autotrader Sync
Status Dashboard ▼
What it does
The VPS (a Digital Ocean server at IP 159.65.17.84) runs a sync script every 30 minutes that pulls the latest vehicle stock from the Autotrader API and pushes it into Sanity. This is what keeps the BMC website stock up to date. The dashboard checks that both the VPS and the Autotrader API are reachable.
What to look out for
Always amber until go live — Sandbox mode
Currently syncing from the Autotrader sandbox (test) API, not the live feed. Vehicle counts may not match the real Autotrader portal. This is expected pre-launch.
Red — VPS unreachable
The sync will have stopped. Vehicles on the BMC site will go stale — prices, stock levels and new arrivals won't update. Contact Elias and check the DigitalOcean dashboard. The VPS may need restarting.

Fuel Finder
Status Dashboard ▼
What it does
A Vercel cron job runs every 30 minutes and calls the UK Government Fuel Finder API to get the current petrol and diesel prices at Brooke Garage. It stores the prices in Sanity so they display on the BMC website petrol stations page. The dashboard checks that the Fuel Finder API is reachable and the cron endpoint is live.
What to look out for
Green — prices updating correctly
Fuel prices on the website are current. No action needed.
Amber — intermittent failures
The Government API is unreliable and times out frequently. The prices on the site may be a few hours old. If the site is showing prices that are clearly outdated (more than 24 hours old) contact Elias to investigate the timeout issue.
Red — station not found or persistent failure
The Brooke Garage station ID may have changed in the Government feed. Check the Vercel logs and contact Elias. The last known prices will remain on the site until the sync recovers.
Known issue — intermittent failuresThe Government Fuel Finder API is unreliable and frequently times out or returns HTTP 503 errors. This is outside BMC's control. Elias is investigating improvements to the timeout handling. Expect to see this card amber frequently.

Resend Emails
Status Dashboard ▼
What it does
Resend is the email delivery service used to send transactional emails — enquiry confirmations to customers and notification emails to the sales team when someone submits a vehicle enquiry or completes a Stripe reservation. The dashboard checks that the Resend API is reachable and the /api/enquiry endpoint is live.
What to look out for
Always amber until go live — Sandbox mode
In sandbox mode emails can only be sent to anthewett@hotmail.com. This is a Resend restriction until the brookemotcentre.co.uk domain is verified. No action needed pre-launch.
Red after go live — emails not delivering
Sales team will not receive enquiry or reservation notifications. Customers won't get confirmation emails. Check the Resend dashboard for bounce or error reports and contact Elias.

CookieYes
Status Dashboard ▼
What it does
CookieYes is the GDPR cookie consent banner that appears when someone visits the BMC website for the first time. It must load before Google Analytics and the Meta Pixel — CookieYes blocks those tracking tools until the visitor accepts cookies. The dashboard checks that the CookieYes CDN script is reachable and the dashboard is accessible.
What to look out for
Amber — script not yet installed on site
The CookieYes script has not been added to the website yet. Brief has been sent to Elias. This must be resolved before go live — without it the site is not GDPR compliant.
Red after installation — CDN unreachable
The cookie banner won't appear and GA4/Meta Pixel may fire without consent, which is a GDPR violation. Contact CookieYes support at cookieyes.com.
Action required — not yet installedCookieYes has been set up and the script tag is ready but Elias has not yet added it to the website. This is a pre-launch blocker and must be completed before the site goes live.

Website Page Load Speed
Metrics Dashboard ▼
What it does
Measures how quickly the BMC website responds to requests. It times three key pages — Homepage, Inventory and Finance — and shows the fastest, average and slowest response times. The speed bars fill up as response time increases — a shorter bar means a faster page.
What good looks like
Under 400ms — Excellent
Pages are loading very quickly. Customers will have a smooth experience.
400ms–1000ms — Acceptable
Slightly slower than ideal but still within acceptable range. Monitor over time to see if it improves or worsens.
Over 1000ms — Slow
Pages are loading too slowly. This can affect Google search rankings and cause customers to leave before the page loads. Contact Elias to investigate — may be a Vercel issue or a slow Sanity query.
🗄
Sanity API Response Time
Metrics Dashboard ▼
What it does
Measures how quickly the Sanity CMS responds to content queries. Every vehicle page, promotion and location on the BMC website requires a Sanity API call. The panel also shows your current Sanity usage against the plan limits — API requests, CDN requests and document count.
API Requests
55.2k of 250k monthly limit — currently at 22%. Plenty of headroom.
CDN Requests
26.9k of 1m monthly limit — very low. No concern.
Documents
595 of 25k limit — vehicles, locations, settings etc. Plenty of room.
What to look out for
Under 500ms — Good
Sanity is responding quickly. Vehicle pages will load fast.
API usage approaching 80% of limit
Consider upgrading the Sanity plan or optimising queries to reduce API calls. Contact Elias.

Stripe Reservations
Metrics Dashboard ▼
What it does
Shows the number of vehicle reservations completed via the £99 Stripe deposit system, total revenue from deposits, and a transaction table showing recent reservations. Use the Today / 7 days / 30 days toggle to see different time periods.
What to look out for
Data is simulated until go live
Real transaction data requires server-side Stripe API access which is not yet connected. Numbers shown are illustrative. After go live connect the Stripe API for real transaction data.
Success rate should always be 100%
Stripe only fires the webhook and completes a reservation when payment is fully confirmed. If success rate drops below 100% after go live, check the Stripe dashboard for failed or disputed payments.

Vehicle Inventory Count
Both Dashboards ▼
What it does
Pulls live vehicle counts directly from Sanity. Shows total live stock, how many are reserved, and a split between Brooke and Norwich branches. Also shows how many vehicles are flagged as Great Value by Autotrader's price indicator. This data is live and accurate.
What to look out for
Count matches your Autotrader portal
If the vehicle count on the dashboard matches what you see in the Autotrader dealer portal, the sync is working correctly.
Count significantly lower than Autotrader
The Autotrader sync may have stopped or be running in sandbox mode. Check the VPS and Autotrader sync status on Dashboard 1.
Reserved count increasing
Good news — more vehicles are being reserved via the website. Remember to manually update those vehicles to Sale in Progress in the Autotrader dealer portal so they also come off the Autotrader listing.

Fuel Price Sync Success Rate
Metrics Dashboard ▼
What it does
Shows the current fuel prices stored in Sanity for Brooke Garage (unleaded, premium, diesel) and how reliably the sync is running. The prices shown are pulled live from Sanity so they reflect exactly what the website is displaying to customers.
What to look out for
Prices match the forecourt
The sync is working. Prices on the BMC website match what customers see at the pump.
Prices are outdated or incorrect
You can manually update prices in Sanity Studio (bmc-vehicles.sanity.studio → Locations → Brooke Garage) while the sync issue is investigated. This is a temporary fix.
Success rate is currently ~15%The Government Fuel Finder API is highly unreliable. Despite frequent failures the last successfully synced prices remain on the site. You can always manually update prices in Sanity if needed.
